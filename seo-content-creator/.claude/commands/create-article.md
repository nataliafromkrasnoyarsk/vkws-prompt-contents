# /seo-content-creator:create-article

Запускает интерактивный пошаговый wizard для создания SEO-статьи с возможностью доработки и гибким сохранением результата.

## Usage

```
/seo-content-creator:create-article
```

Или просто напишите агенту:
- "Создай статью"
- "Напиши статью"
- "Помоги написать SEO-статью"
- "Запусти wizard"
- "Новая статья"

## Wizard Steps

Команда запускает интерактивный процесс из 11 шагов:

```
┌─────────────────────────────────────────────────────────┐
│  STEP 1:  Тема статьи                        [ОБЯЗАТ.] │
│  ↓                                                      │
│  STEP 2:  Выбор эксперта                     [ОБЯЗАТ.] │
│  ↓                                                      │
│  STEP 3:  Ваша информация по теме            [ОПЦИОН.] │
│  ↓                                                      │
│  STEP 4:  Ключевые слова                     [ОБЯЗАТ.] │
│  ↓                                                      │
│  STEP 5:  Режим генерации                    [ОБЯЗАТ.] │
│  ↓                                                      │
│  STEP 6:  Тон и аудитория                    [ОБЯЗАТ.] │
│  ↓                                                      │
│  STEP 7:  Структура статьи                   [ОПЦИОН.] │
│  ↓                                                      │
│  STEP 8:  Дополнительные настройки           [ОПЦИОН.] │
│  ↓                                                      │
│  STEP 9:  Подтверждение и генерация                     │
│  ↓                                                      │
│  STEP 10: Доработка результата               [ОПЦИОН.] │
│  ↓                                                      │
│  STEP 11: Сохранение                         [ОПЦИОН.] │
└─────────────────────────────────────────────────────────┘
```

## Navigation Commands

В любой момент можно использовать:

| Команда | Действие |
|---------|----------|
| `назад` | Вернуться на шаг назад |
| `пропустить` | Пропустить опциональный шаг |
| `статус` | Показать текущие параметры |
| `начать заново` | Перезапустить wizard |
| `помощь` | Показать справку |
| `отмена` | Отменить и выйти |

## Quick Start Options

Можно сразу передать некоторые параметры:

```
/seo-content-creator:create-article --topic "Моя тема"
```

```
/seo-content-creator:create-article --quick
```

Флаг `--quick` использует значения по умолчанию для опциональных шагов.

## Parameters Reference

### Step 1: Тема
- **Тип:** строка
- **Обязательно:** да
- **Пример:** "Как выбрать CRM для малого бизнеса"

### Step 2: Выбор эксперта
- **Тип:** выбор из списка
- **Обязательно:** да
- **Варианты:**
  - `vk-cloud-security` — Эксперт по ИБ VK Cloud
  - `general` — Универсальный автор (по умолчанию)

### Step 3: Ваша информация по теме
- **Тип:** текст или "пропустить"
- **Обязательно:** нет
- **Описание:** Факты, статистика, цитаты, которые нужно включить в статью
- **Пример:** "У нас 152-ФЗ сертификация, 500+ клиентов в финтехе"

### Step 4: Ключевые слова
- **Тип:** список или "авто"
- **Обязательно:** да
- **Пример:** "crm система, выбор crm, crm 2024"

### Step 5: Режим
- **Тип:** выбор из списка
- **Обязательно:** да
- **Варианты:**
  - `balanced` — стандартный (по умолчанию)
  - `deep_expert` — технический экспертный
  - `light` — упрощённый
  - `strict_seo` — максимум SEO
  - `brand_heavy` — акцент на бренд

### Step 6: Тон и аудитория
- **Тип:** два выбора из списка
- **Обязательно:** да
- **Тон:** expert, neutral, technological
- **Аудитория:** SMB, enterprise, developers, marketers, general

### Step 7: Структура
- **Тип:** список разделов или "авто"
- **Обязательно:** нет
- **По умолчанию:** автогенерация

### Step 8: Дополнительные настройки
- **Обязательно:** нет
- **Включает:**
  - Длина (1500/2500/3500 слов)
  - Ограничения (no_promo, no_comparison, etc.)
  - Работа с фактами (strict/moderate/flexible)
  - Уникальность (1/2/3)

### Step 10: Доработка результата
- **Тип:** выбор из списка
- **Обязательно:** нет
- **Варианты:**
  - `всё отлично` — Перейти к сохранению
  - `изменить тон` — Перегенерировать в другом тоне
  - `изменить аудиторию` — Перегенерировать для другой аудитории
  - `короче` — Сократить статью
  - `подробнее` — Расширить статью
  - `упростить` — Сделать проще для понимания
  - `добавить примеры` — Добавить больше примеров
  - `свои правки` — Указать конкретные изменения

### Step 11: Сохранение результата
- **Тип:** выбор из списка
- **Обязательно:** нет
- **Варианты:**
  - `markdown` — Сохранить в .md файл
  - `html` — Сохранить в .html файл
  - `content-hub` — Сохранить в content-hub/drafts/
  - `clipboard` — Скопировать в буфер обмена
  - `все` — Сохранить во все форматы
  - `не сохранять` — Пропустить (по умолчанию)

```json

  "meta_description": "Мета-описание (150-160 символов)",

  "h1_variants": ["Вариант 1", "Вариант 2", "Вариант 3"],
  "article_full_text": "Полный текст в Markdown",

  "editor_notes": ["Заметка 1", "Заметка 2"],

```

## Examples

### Минимальный запуск
```
/seo-content-creator:create-article
```
Запустит полный wizard с вопросами.

### С предзаданной темой
```
/seo-content-creator:create-article --topic "Docker для начинающих"
```
Пропустит шаг 1, начнёт с выбора эксперта.

### Быстрый режим
```
/seo-content-creator:create-article --quick --topic "Kubernetes vs Docker Swarm"
```
Запросит только обязательные параметры, остальное по умолчанию.
