## ТИПЫ МЕРОПРИЯТИЙ

### Вебинар (online)

```yaml
characteristics:
  duration: 1-2 часа
  format: online
  audience_size: 50-500
  
structure:
  - О мероприятии (контекст + темы)
  - Спикеры (2-4 человека)
  - Q&A секция
  - Запись трансляции
  
example_title: "VK Process Mining 2.0: новые возможности"

required_blocks:
  - header
  - title
  - description
  - speakers
  - registration_cta
  
optional_blocks:
  - program (если >1 часа)
  - partners
  - aftermovie
```

### Конференция (offline/hybrid)

```yaml
characteristics:
  duration: 4-8 часов
  format: offline или hybrid
  audience_size: 100-1000
  
structure:
  - О мероприятии
  - Полная программа с таймингами
  - Панельные сессии
  - Доклады
  - Клиентские кейсы
  - Кофе-брейки и нетворкинг
  - Партнёры (генеральные, технологические, медиа)
  
example_title: "VK Cloud Conf 2024"

required_blocks:
  - header
  - title
  - description
  - program (детальная)
  - speakers
  - partners
  
optional_blocks:
  - aftermovie
  - photo_gallery
```

### Митап (offline)

```yaml
characteristics:
  duration: 2-4 часа
  format: offline
  audience_size: 30-100
  informal: true

structure:
  - О мероприятии
  - Краткая программа
  - Спикеры
  - Нетворкинг

example_title: "Backend Meetup в Ярославле"

required_blocks:
  - header
  - title
  - description
  - speakers
```

### Хакатон (hybrid)

```yaml
characteristics:
  duration: 24-72 часа
  format: hybrid
  audience_size: 50-500
  competitive: true

structure:
  - О мероприятии (тема, призы)
  - Расписание (старт, чекпоинты, финал)
  - Треки/номинации
  - Жюри
  - Призы
  - Партнёры

example_title: "VK Hackathon: AI Solutions"

required_blocks:
  - header
  - title
  - description
  - schedule
  - tracks
  - jury
  - prizes

optional_blocks:
  - partners
  - aftermovie
  - winner_projects
```

### Демо-день (hybrid)

```yaml
characteristics:
  duration: 2-4 часа
  format: hybrid
  audience_size: 50-200
  showcase: true

structure:
  - О мероприятии
  - Программа презентаций
  - Проекты/команды
  - Жюри/эксперты

example_title: "Demo Day: Выпуск акселератора VK"

required_blocks:
  - header
  - title
  - description
  - projects

optional_blocks:
  - jury
  - partners
  - aftermovie
```

## ШАБЛОН "О МЕРОПРИЯТИИ"

### Структура

```markdown
## О мероприятии

{Контекст: почему это актуально, что изменилось, зачем нужно. 2-4 предложения.}

На {тип} **{дата}** в **{время}** (МСК) {что_произойдёт}.

**В программе:**

- {Тема 1}
- {Тема 2}
- {Тема 3}
- Q&A

**Спикеры:**

- {Имя Фамилия}, {должность}, {компания}
- {Имя Фамилия}, {должность}, {компания}

{Тип} будет интересен {целевая_аудитория}.

До встречи на {тип}!
```

```markdown
## О мероприятии

доступен не только в коробочном, но и в облачном решении.

На вебинаре **22 октября** в **11:00** (МСК) эксперты расскажут, 
как применять VK Process Mining 2.0 для внутреннего аудита.

**В программе:**

- Что нового в версии 2.0
- Как работает мультипроцессная аналитика
- Как проходит проверка процессов на соответствие эталонным моделям
- Под какие нужды подходят локальная и облачная версии продукта
- Q&A

**Спикеры:**

- Александр Вилкул, старший менеджер продукта VK Tech
- Глеб Берюков, менеджер продукта VK Tech

Вебинар будет интересен внутренним аудиторам, которым важно 
анализировать и улучшать бизнес-процессы.

До встречи на вебинаре!
```

```markdown

- Тезис 2
- Тезис 3
```

### Типы блоков

| Тип | Оформление | Описание |
|-----|------------|----------|
| Регистрация | #### Регистрация | Без спикера |
| Открытие | #### Открытие конференции | Приветственное слово |
| Доклад | Доклад + #### | Основной формат |
| Панельная сессия | Панельная сессия + #### | Несколько спикеров |
| Клиентский кейс | Клиентский кейс + #### | Выступление клиента |
| Кофе-брейк | #### Кофе-брейк | Без спикера |
| Обед | #### Обед | Без спикера |
| Завершение | #### Завершение | Нетворкинг |

## СТИЛЬ И ТОНАЛЬНОСТЬ

### Правила текста

```yaml
tone: деловой, но дружелюбный

do:
  - Использовать "мы", "вы"
  - Писать конкретно (даты, цифры, имена)
  - Показывать ценность для участника
  - Добавлять призыв "До встречи!"

dont:
  - Излишне формальный язык
  - Общие фразы без конкретики
  - Рекламные преувеличения
  - Много восклицательных знаков
```

### Формат даты

```yaml
full: "22 октября 2024 года"
short: "22.10.2024"
header: "22 октября 2024, 11:00"
```

### Формат времени

```yaml
format: "ЧЧ:ММ (МСК)"
examples:
  - "11:00 (МСК)"
  - "17:00"
range: "10:00 - 10:30"
```

## ЧЕКЛИСТ ГОТОВНОСТИ

```yaml
required:
  - [ ] Тип мероприятия указан
  - [ ] Название информативное
  - [ ] Дата и время указаны
  - [ ] Формат (online/offline) указан
  - [ ] Описание содержит контекст
  - [ ] Список тем/программа есть
  - [ ] Хотя бы 1 спикер указан
  
quality:
  - [ ] Тезисы докладов конкретные
  - [ ] Целевая аудитория определена
  - [ ] CTA для регистрации есть
  - [ ] Нет грамматических ошибок
  
optional:
  - [ ] Партнёры добавлены
  - [ ] Aftermovie/трансляция
  - [ ] Соцсети для подписки
```
