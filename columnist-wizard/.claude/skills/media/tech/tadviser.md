# TAdviser — Руководство по стилю

## Профиль издания

| Параметр | Значение |
|----------|----------|
| URL | tadviser.ru |
| Аудитория | CIO, ИТ-директора, аналитики, enterprise |
| Объём | Обзорные статьи: 4,000-8,000+ слов; Экспертные цитаты: 200-500 слов |
| Технический уровень | 40-60% |
| Тон | Аналитический, business-IT |
| Формат | Обзоры с экспертными мнениями, wiki-модель |
| Контакты | news@tadviser.ru, editor@tadviser.ru |

## Особенности TAdviser

### Формат публикаций
- Комплексные обзоры рынков и технологий
- Экспертные мнения встроены в статьи (не отдельные колонки)
- Wiki-формат: статьи редактируемы и дополняемы
- Фокус на российском enterprise IT
- Интеграция с базой компаний и проектов

### Типичная структура обзора
```
Обзор: [Тема] в России 2025
├── Введение и методология
├── Объём рынка и динамика
├── Ключевые тренды
│   ├── Тренд 1 + экспертные мнения
│   ├── Тренд 2 + экспертные мнения
│   └── Тренд 3 + экспертные мнения
├── Барьеры и вызовы
├── Прогнозы
└── Заключение
```

## Форматы участия

### 1. Экспертная цитата в обзоре

**Формат:**
```
[Имя] из «[Компания]» отмечает, что [экспертное мнение].
По словам [Имя], [должность] [Компания], «[прямая цитата]».
```

**Объём:** 100-300 слов на одну цитату (2-4 цитаты в обзоре)

### 2. Интервью для обзора

**Формат:** Ответы на 5-10 вопросов от редакции
**Объём:** 500-1,500 слов суммарно

### 3. Проектный кейс

**Формат:** Описание внедрения в формате TAdviser
**Объём:** 800-1,500 слов

## Структура экспертной цитаты

```
ПОЗИЦИЯ (1-2 предложения)
├── Главный тезис эксперта
└── Связь с темой обзора

АРГУМЕНТАЦИЯ (2-3 предложения)
├── Данные / статистика
├── Пример из практики
└── Обоснование позиции

ВЫВОД / ПРОГНОЗ (1-2 предложения)
├── Что это значит для рынка
└── Рекомендация или прогноз
```

## Тематические области TAdviser

- Искусственный интеллект и ML
- Облачные технологии
- Информационная безопасность
- Цифровая трансформация
- Базы данных и СУБД
- ИТ в отраслях (банки, госсектор, ритейл, промышленность)
- ERP, CRM, документооборот
- Телеком и связь

## Стилистические требования

### Язык
- Формальный деловой регистр
- Аналитический, не рекламный
- Количественные оценки приветствуются
- Ссылки на источники обязательны

### Тональность
- Экспертная, взвешенная
- Признание разных точек зрения
- Forward-looking (прогнозы)
- Vendor-neutral (насколько возможно)

## Примеры экспертных цитат

### Пример 1: Интервью — генеральный директор о стратегии компании

**Контекст:** Интервью для обзора «Рынок корпоративного ПО в России 2025»

**Текст:**

Иван Смирнов, генеральный директор [Компания], в интервью TAdviser подчёркивает, что международные гиперскейлеры не собираются размещать инфраструктуру в России — и это создаёт стратегическое окно возможностей для российских вендоров.

«Российские продуктовые ИТ-компании должны сыграть ключевую роль в развитии рынка. За три года мы прошли путь, на который западным аналогам потребовалось десятилетие. По итогам 2024 года выручка компании выросла на 42% — это показатели, сопоставимые с позициями крупных западных вендоров в России до 2022 года», — отмечает Смирнов.

По его словам, компании удаётся сочетать «инженерное совершенство интернет-компании с процессной дисциплиной enterprise-вендора», избегая «конвейера авралов».

**Оценка:** Соответствует формату TAdviser. Стратегическое позиционирование, конкретные цифры, взгляд топ-менеджера.

### Пример 2: CISO о безопасности облака

**Контекст:** Обзор «Информационная безопасность облачных сервисов 2025»

**Текст:**

Алексей Козлов, директор по информационной безопасности [Компания], объясняет, почему принцип Defense in Depth остаётся фундаментом защиты облачной инфраструктуры.

«Мы выстраиваем многоуровневую защиту: физическая безопасность ЦОД, сетевая защита от DDoS, безопасность операционной системы, защита на уровне приложений. Каждый уровень независим — отказ одного не компрометирует систему», — поясняет эксперт.

Платформа обрабатывает миллионы событий безопасности в секунду. Получение сертификата ФСТЭК России позволяет клиентам строить инфраструктуру для работы с персональными данными и критичными системами.

«Главные угрозы — не хакеры, а фишинг и ошибки конфигурирования. Облачная платформа обеспечивает технологический базис защиты, но клиент отвечает за уровень приложений», — добавляет Козлов.

**Оценка:** Отличный пример для TAdviser. Техническая глубина, конкретные метрики, compliance-контекст.

### Пример 3: Директор по данным о платформах данных

**Контекст:** Обзор «Data Lakehouse и платформы данных в России 2025»

**Вопрос редакции:** Почему компании переходят от Data Lake к Data Lakehouse?

**Ответ:**

По словам Сергея Петрова, директора по данным [Компания], архитектурные паттерны Data Mesh и LakeHouse позволяют лучше справляться с растущими объёмами и разнообразием данных.

«Data Lake решал задачу хранения, но не аналитики. Данные в сыром виде непригодны для ежедневной работы в BI-системах, но могут использоваться для проверки гипотез с помощью ML/AI. LakeHouse объединяет оба сценария», — объясняет эксперт.

Платформа обслуживает клиентов с петабайтами данных и высокой пропускной способностью. Включает S3-совместимое хранилище, Data Fabric для работы с метаданными и MLOps-сервисы.

«Рынок платформ данных в России продолжит расти. Выиграют те, кто предложит полный цикл: от хранения до аналитики и машинного обучения в едином интерфейсе», — прогнозирует Петров.

**Оценка:** Соответствует формату TAdviser. Технический тренд, данные о масштабе, рыночный прогноз.

### Пример 4: CPO о продуктовом развитии

**Контекст:** Обзор «Managed Services и PaaS в России 2025»

**Текст:**

Михаил Волков, директор по продукту [Компания], отмечает сдвиг в структуре потребления облачных услуг от IaaS к PaaS.

«Выручка направления Managed Services показывает значительный рост год к году. Клиенты предпочитают managed-базы данных, Kubernetes, Kafka — не потому что не умеют настраивать сами, а потому что стоимость содержания команды DBA превышает стоимость сервиса в 2-3 раза», — говорит эксперт.

Managed Kubernetes востребован значительной долей клиентов. Компания активно развивает новые продукты: Cloud Desktop для распределённых команд, Data Platform для работы с большими данными.

«Гибридность — не компромисс, а осознанный выбор. On-Cloud + On-Premise позволяет клиентам балансировать между контролем и гибкостью», — подчёркивает Волков.

**Оценка:** Хороший пример для TAdviser. Продуктовый фокус, конкретные метрики, позиционирование.

## Как работать с TAdviser

### Подача материала
1. Связаться с редакцией: news@tadviser.ru или editor@tadviser.ru
2. Предложить экспертизу по конкретной теме
3. Дождаться запроса на комментарий или интервью
4. Подготовить цитаты в требуемом формате

### Для wiki-публикаций
- Зарегистрироваться на портале
- Добавлять информацию в профиль компании
- Публиковать проектные кейсы
- Обновлять данные о продуктах

### Платные опции
- Гарантированная публикация
- Размещение в топе категории
- Расширенный профиль компании

## Чеклист для экспертной цитаты

- [ ] Связь с темой обзора
- [ ] Чёткая позиция (не «с одной стороны, с другой стороны»)
- [ ] Данные или примеры в поддержку
- [ ] Прогноз или рекомендация
- [ ] Корректное указание должности
- [ ] Объём 100-300 слов
- [ ] Нет прямой рекламы продуктов
- [ ] Vendor-neutral тон (насколько возможно)
