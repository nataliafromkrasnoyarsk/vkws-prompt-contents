## Tech Content Creator

**Расположение:** `plugins/tech-content-creator/`

Профессиональная система создания контента для высокотехнологичных компаний. Полный цикл от исследования трендов до публикации статей в ведущих бизнес и технологических медиа.

### Специализированные агенты

| Агент | Роль | Описание |
|-------|------|----------|
| **Tech Content Strategist** | Стратег контента | Исследование трендов, конкурентный анализ, IMPACT-скоринг тем, планирование контент-стратегии, подготовка питчей для изданий |
| **Tech Content Writer** | Технический писатель | Создание opinion pieces, технических туториалов, кейс-стади, аналитических статей. Двуязычное написание (RU/EN) |
| **Tech Fact-Checker** | Фактчекер | Проверка статистики и технических утверждений, валидация источников, тестирование примеров кода, верификация цитат |
| **Content Editor** | Главный редактор | Структурное редактирование, улучшение читаемости, адаптация под конкретные издания, билингвальные стандарты |
| **SEO Content Optimizer** | SEO-оптимизатор | Оптимизация заголовков, создание мета-описаний, стратегия ключевых слов, подготовка для соцсетей |

### Навыки (Skills)

- **tech-trends-research** — методология исследования технологических трендов, анализ рынка, идентификация контент-возможностей
- **enterprise-storytelling** — корпоративное повествование, трансформация сложных технических концепций в compelling narratives
- **technical-writing-standards** — стандарты технического письма для ясного и профессионального контента
- **fact-checking-methodology** — журналистские стандарты проверки фактов, трёхуровневая модель верификации
- **editorial-excellence** — продвинутые техники редактирования для трансформации контента в exceptional articles
- **media-publishing-guidelines** — гайдлайны для публикации в Forbes, TechCrunch, HBR, Habr, VC.ru и других

### Целевые издания

**Глобальные:** Forbes, Harvard Business Review, MIT Technology Review, TechCrunch, Wired, Ars Technica

**Российские:** Ведомости, РБК, Коммерсантъ, Хабр, VC.ru, CNews

## SEO Content Creator

**Расположение:** `seo-content-creator/`

Интерактивный агент для создания SEO-оптимизированных статей с пошаговым заполнением параметров.

### Wizard из 7 шагов

```
┌──────────────────────────────────────────┐
│  Step 1: Тема статьи          [ОБЯЗАТ.] │
│  Step 2: Ключевые слова       [ОБЯЗАТ.] │
│  Step 3: Режим генерации      [ОБЯЗАТ.] │
│  Step 4: Тон и аудитория      [ОБЯЗАТ.] │
│  Step 5: Структура            [ОПЦИОН.] │
│  Step 6: Доп. настройки       [ОПЦИОН.] │
│  Step 7: Подтверждение                  │
└──────────────────────────────────────────┘
```

### Режимы генерации

| Режим | Описание |
|-------|----------|
| `balanced` | Стандартный режим — баланс между SEO и читаемостью |
| `deep_expert` | Технический экспертный контент для профессионалов |
| `light` | Упрощённый контент для широкой аудитории |
| `strict_seo` | Максимальная SEO-оптимизация |
| `brand_heavy` | Акцент на бренд и позиционирование |

### Настройки тона и аудитории

**Тон:** expert (профессиональный), neutral (нейтральный), technological (технический)

**Аудитория:** SMB (малый/средний бизнес), enterprise (крупный бизнес), developers (разработчики), marketers (маркетологи), general (широкая аудитория)

### Результат генерации

- SEO-заголовок (Title) до 60 символов
- Мета-описание (Description) 150-160 символов
- SEO-friendly URL
- 3 варианта H1
- Полный текст статьи в Markdown
- Статистика: количество слов, ключей, LSI
- Заметки для редактора

## Event Page Wizard

**Расположение:** `event-page-wizard/`

Интерактивный агент для создания страниц мероприятий в стиле VK Cloud. Поддерживает вебинары, конференции, митапы, воркшопы, хакатоны и демо-дни.

### Wizard из 10 шагов

```
┌──────────────────────────────────────────────────────────┐
│  Step 1:  Тип мероприятия                     [ОБЯЗАТ.] │
│  Step 2:  Название                            [ОБЯЗАТ.] │
│  Step 3:  Дата и время                        [ОБЯЗАТ.] │
│  Step 4:  Формат и место                      [ОБЯЗАТ.] │
│  Step 5:  Описание                            [ОБЯЗАТ.] │
│  Step 6:  Программа                           [ОПЦИОН.] │
│  Step 7:  Спикеры                             [ОБЯЗАТ.] │
│  Step 8:  Целевая аудитория                   [ОПЦИОН.] │
│  Step 9:  Партнёры                            [ОПЦИОН.] │
│  Step 10: Дополнительно                       [ОПЦИОН.] │
├──────────────────────────────────────────────────────────┤
│  ФИНАЛ:   Превью → Сохранение файлов                    │
└──────────────────────────────────────────────────────────┘
```

### Типы мероприятий

| Тип | Описание | Формат по умолчанию |
|-----|----------|---------------------|
| Вебинар | Онлайн-трансляция с презентацией и Q&A | online |
| Конференция | Масштабное мероприятие с несколькими треками | hybrid |
| Митап | Неформальная встреча сообщества | offline |
| Воркшоп | Практическое занятие с hands-on опытом | offline |
| Хакатон | Соревнование разработчиков | hybrid |
| Демо-день | Презентация проектов и продуктов | hybrid |

### Результат генерации

- **Markdown-страница** (`events/{slug}.md`) — готовая страница мероприятия
- **JSON-структура** (`events/{slug}.json`) — структурированные данные для интеграций

### Параметры запуска

```bash
/create-event                          # Интерактивный режим
/create-event --type вебинар           # С предустановленным типом
/create-event --quick                  # Только обязательные шаги
/create-event --lang en                # Генерация на английском
/create-event --output ./my-events     # Своя директория для файлов
```

## YAML-промты

**Расположение:** `prompts/`

Standalone промты для использования в любых LLM без Claude Code.

### prompt_seo_1_analyze_and_build_params.yaml

**Аналитик → Архитектор параметров**

Анализирует входные данные (тема, план, ключевые слова, позиционирование, тональность) и формирует полный набор параметров для генерации статьи.

**Входные параметры для настройки:**
- `MODE` — режим генерации
- `RISK_PROFILE` — профиль риска
- `STRUCTURE_LEVEL` — уровень структурирования
- `UNIQUENESS_LEVEL` — уровень уникальности
- `FACT_SOURCE_BEHAVIOR` — поведение с источниками
- `COVERAGE_DEPTH` — глубина раскрытия
- `SERP_COMPETITOR_LOGIC` — логика анализа конкурентов

**Автоматически определяет:** целевую аудиторию, LSI-ключи, улучшения структуры.

**Выход:** `FINAL_GENERATION_PARAMS` — полный набор параметров для генерации.

### prompt_seo_2_generate_article.yaml

**Генератор статей**

Принимает параметры из первого промта и создаёт SEO-оптимизированную статью.

### prompt_seo_3_infostyle_transformer_2_0.yaml

**Инфостайл-трансформер для лендингов и презентаций**

Редактура и SEO-оптимизация текстов без изменения структуры. Все заголовки, порядок блоков и параграфы сохраняются — переписывается только содержание.

**Возможности:**
- Улучшение стиля, сокращение воды, удаление дублей
- Адаптация тона (expert / neutral / corporate)
- Встраивание ключевых слов
- Создание метатегов
- Оптимизация под нейропоиск (SGE / Y1 / AI Overviews)

## Доступные команды

Slash-команды для Claude Code:

| Команда | Описание |
|---------|----------|
| `/article-workflow` | Полный workflow создания статьи: исследование → идея → черновик → фактчек → редактура → SEO |
| `/seo-create-article` | Интерактивный SEO wizard с пошаговым заполнением параметров |
| `/draft-article` | Создание черновика статьи по content brief или теме |
| `/trend-research` | Исследование технологических трендов и контент-возможностей |
| `/create-event` | Интерактивный wizard для создания страниц мероприятий |
