# Zen Article Wizard — VK WorkSpace

Запусти интерактивный визард для создания статьи о VK WorkSpace для канала в Яндекс Дзен, оптимизированной под алгоритм рекомендаций.

## Инструкции

1. Загрузи скиллы:
   - `zen-knowledge` — база знаний о платформе Дзен: алгоритм, форматы, оптимизация дочитываний, формулы заголовков, чеклист качества
   - `vk-workspace-knowledge` — база знаний о VK WorkSpace: продукты, позиционирование, целевые аудитории, конкуренты, терминология

2. Действуй как агент `zen-article-wizard` — проведи пользователя через 11 шагов:
   - Шаг 1: Тип статьи (ОБЯЗАТЕЛЬНО) — образовательная, гайд, сравнение, кейс, новость, подборка, тренд, мнение
   - Шаг 2: Тема и угол (ОБЯЗАТЕЛЬНО) — кластер + конкретная тема
   - Шаг 3: Продукт VK WorkSpace (ОБЯЗАТЕЛЬНО) — Суперапп, Мессенджер+ВКС, Проекты+Доска, Почта+Календарь+Диск
   - Шаг 4: Упоминания VK WorkSpace (ОБЯЗАТЕЛЬНО) — макс 2-4 через сценарии / без лимита
   - Шаг 5: Целевая аудитория (ОБЯЗАТЕЛЬНО) — широкая, руководители СМБ, ИТ, менеджеры
   - Шаг 6: Дополнительная информация (ОПЦИОНАЛЬНО) — факты, кейсы, цитаты, ссылки
   - Шаг 7: Режим генерации (ОБЯЗАТЕЛЬНО) — Engagement, Expert, Light, Story
   - Шаг 8: Заголовок (ОБЯЗАТЕЛЬНО) — 3 варианта по формулам Дзен
   - Шаг 9: Подтверждение параметров
   - Шаг 10: Генерация → ВЫВОД ПОЛНОГО ТЕКСТА В ДИАЛОГ → самопроверка → сохранение файла
   - Шаг 11: Цикл доработок (полный текст в диалог после каждой правки) → коммит и пуш ТОЛЬКО после явного «доработки не нужны»

3. На каждом обязательном шаге используй `AskUserQuestion` для выбора параметров.

4. При генерации статьи соблюдай:
   - **Объём: 2500-3500 символов без пробелов** (допуск ±10-20%)
   - **Структура: не более 5 заголовков H2** на статью
   - **Заголовок из ТЗ: если тема указана в ТЗ — использовать дословно**, крайне нежелательно менять
   - **Поисковые запросы: если указаны — каждый минимум 1 раз**, равномерно по тексту, по возможности в H2
   - Оптимизация: дочитывания > CTR > вовлечение > cite-safety (AI) > SEO-ключи
   - **Cite-Safety (AI-поисковики):** ключевые блоки (лид, тезисы, рекомендации, выводы) писать как Extractable Trust Blocks — самодостаточные при вырывании из контекста. Формула: утверждение → объяснение → граница применимости → пример/источник. Нет «он/это/данный» без пояснения, нет категоричности без границ.
   - Формат: короткие абзацы (1-3 предложения), интригующие подзаголовки
   - Стиль: разговорный, живой, с конкретикой и примерами
   - Удержание: открытые петли
   - Визуал: отмечай места для 3-5 изображений
   - VK WorkSpace: упоминания по режиму из Шага 4 (2-4 или без лимита), без промо в обоих случаях
   - CTA: вопрос для комментариев + призыв к подписке

5. Сохраняй статьи в `content-hub/zen-articles/drafts/{YYYY-MM-DD}-{slug}.md`

6. **Порядок после генерации:** выведи полный текст в диалог → самопроверка → спроси о доработках → цикл правок (полный текст после каждой) → коммит/пуш ТОЛЬКО после явного «доработки не нужны». НЕ предлагай коммит до этого момента.

## Аргумент $ARGUMENTS

Если передан аргумент, используй его как тему статьи и **как заголовок** (крайне нежелательно менять тему из ТЗ). Пропусти Шаг 2 (определение конкретной темы) и Шаг 8 (генерация заголовка — используй тему как есть). Шаг 1 (тип статьи) всё равно нужно пройти.
