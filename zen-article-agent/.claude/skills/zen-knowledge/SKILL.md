# База знаний Дзен для статей

## 1. О платформе Дзен

**Дзен** — контент-платформа Яндекса (с 2022 — VK), крупнейшая рекомендательная площадка в Рунете. Алгоритм подбирает контент на основе интересов пользователя.

### Ключевые факты
- Более 70 млн уникальных пользователей в месяц (MAU)
- Интегрирован в экосистему VK (с 2022)
- Основной трафик — мобильные устройства (70%+)
- Алгоритмическая лента — главный источник трафика (не поисковый)
- Монетизация: партнерская программа, нативная реклама, подписки
- Контент: статьи, видео, короткие видео, посты, нарративы (stories)
- Аудитория: 25-55 лет, широкий профиль, преимущественно Россия и СНГ

### Ключевое отличие от SEO-статей
- **SEO** — оптимизация под поисковые запросы (Яндекс/Google)
- **Дзен** — оптимизация под алгоритм рекомендаций (вовлечение, дочитывания, время на странице)
- В Дзене ключевые слова вторичны — первичны поведенческие метрики
- Заголовок и обложка решают судьбу статьи (CTR в ленте)

## 2. Алгоритм Дзен — как работает рекомендация

### 2.1. Воронка показа

```
Показ в ленте → Клик (CTR) → Чтение → Дочитывание → Реакция → Повторный визит
```

### 2.2. Ключевые метрики алгоритма

| Метрика | Описание | Целевое значение |
|---------|----------|-----------------|
| **CTR** | Кликабельность в ленте | > 3-5% |
| **Дочитывания** | % читателей, дочитавших до конца | > 50-60% |
| **Время чтения** | Среднее время на статье | > 2-3 мин |
| **Вовлечение** | Лайки, комментарии, репосты | Чем больше, тем лучше |
| **Карма канала** | Общий рейтинг автора | Влияет на охваты |
| **Негатив** | Жалобы, «не интересно» | Минимум |

### 2.3. Логика алгоритма

1. **Первичный показ** — статья показывается малой аудитории (~100-500 чел.)
2. **Оценка CTR** — если кликают > 3%, расширяет аудиторию
3. **Оценка дочитываний** — если > 50% дочитывают, расширяет ещё
4. **Вовлечение** — лайки/комменты дают дополнительный буст
5. **Расширение** — статья попадает в ленты всё большего числа пользователей
6. **Затухание** — через 24-72 часа охваты падают (если нет вирусного роста)

### 2.4. Факторы ранжирования

**Положительные:**
- Высокий CTR заголовка + обложки
- Высокий % дочитываний
- Длительное время на странице
- Лайки, комментарии, репосты
- Подписки после прочтения
- Регулярность публикаций (2-5 статей в неделю)
- Качественный контент без кликбейта

**Отрицательные:**
- Низкий CTR (< 1%) — статья «умирает»
- Низкие дочитывания (< 30%) — алгоритм снижает показы
- Жалобы и «не интересно»
- Кликбейт (несоответствие заголовка содержанию)
- Нарушение правил платформы
- Длительные перерывы в публикациях
- Копипаст и неуникальный контент

## 3. Форматы контента

### 3.1. Статья (основной формат)

**Оптимальный объём:** 2500-3500 символов без пробелов (допустимо отклонение 10-20%)

> **Единица измерения — символы без пробелов.** Это примерно 400-550 слов. Компактный формат: каждое слово на счету.

**Структура:**
- Заголовок: если из ТЗ — использовать дословно; если свободный — до 56 символов (видимая часть в ленте)
- Описание (подзаголовок): до 200 символов
- Обложка: яркая, читаемая, контрастная
- Лид: 2-3 предложения — крючок
- Тело: **не более 5 заголовков H2** (учитывая компактный объём)
- Визуал: 3-5 изображений на статью
- Заключение: выводы + призыв к действию (подписка, комментарий)

**Правило заголовка из ТЗ:** Если в ТЗ (техническом задании) указана тема — использовать её как заголовок статьи. Крайне нежелательно менять.

### 3.2. Пост (короткий формат)

**Объём:** до 1500 символов
**Применение:** быстрые мысли, инфоповоды, провокации для обсуждения

### 3.3. Видео

**Длительность:** 3-15 минут (оптимально 5-8 мин)
**Применение:** обзоры, инструкции, демонстрации

### 3.4. Галерея / Нарратив

**Формат:** серия слайдов с текстом и картинками
**Применение:** подборки, чек-листы, сравнения

## 4. Правила написания заголовков для Дзен

### 4.1. Формулы заголовков

| Формула | Пример | Когда использовать |
|---------|--------|-------------------|
| **Число + ценность** | «7 ошибок при переходе на российское ПО» | Списки, подборки |
| **Как + результат** | «Как мы перевели 500 человек с Outlook на отечественную почту за 2 недели» | Гайды, кейсы |
| **Проблема + решение** | «Сотрудники саботируют новое ПО? 5 тактик, которые работают» | Боль аудитории |
| **Вопрос** | «Можно ли заменить Jira российским аналогом без боли?» | Дискуссии |
| **Противопоставление** | «Мессенджер для работы: Telegram vs корпоративные решения» | Сравнения |
| **Провокация** | «Импортозамещение ПО провалилось? Разбираем реальные цифры» | Мнения |
| **Было-стало** | «До: 5 сервисов. После: один суперапп. Как изменилась работа команды» | Кейсы |
| **Секрет / инсайд** | «Что не говорят вендоры: скрытые расходы миграции на On-Premises» | Экспертный контент |
| **Личный опыт** | «Я внедрил ВКС для 300 человек. Рассказываю, что пошло не так» | Сторителлинг |
| **Тренд + год** | «Корпоративные коммуникации 2026: 5 трендов, которые нельзя игнорировать» | Тренд-статьи |

### 4.2. Правила заголовков

**ДЕЛАЙ:**
- Конкретика: числа, сроки, названия
- Польза: что получит читатель
- Интрига: но не обманывай ожидания
- Эмоция: проблема, удивление, вызов
- Длина: 40-56 символов (видимая часть в мобильной ленте)

**НЕ ДЕЛАЙ:**
- Кликбейт — несоответствие заголовка содержанию (алгоритм наказывает)
- Caps Lock — ЗАГОЛОВОК КАПСОМ
- Восклицательные знаки — «Невероятно!!!»
- Абстрактные формулировки — «Интересные факты о...»
- Слишком длинные — обрезаются в мобильной ленте

### 4.3. Описание (подзаголовок)

- До 200 символов
- Дополняет, а не дублирует заголовок
- Раскрывает ценность статьи
- Может содержать целевое действие: «Читайте, чтобы не повторить чужих ошибок»

## 5. Оптимизация дочитываний

### 5.1. Структура для максимальных дочитываний

```
1. КРЮЧОК (первые 2-3 предложения)
   → Боль, вопрос, провокация, шокирующий факт
   → Зачем читать дальше

2. ОБЕЩАНИЕ (абзац 2)
   → Что получит читатель
   → Краткий анонс структуры

3. ОСНОВНАЯ ЧАСТЬ
   → Короткие абзацы (2-3 предложения)
   → Подзаголовки-зацепки (интригуют, а не просто называют тему)
   → Визуал каждые 300-500 слов
   → Примеры с конкретикой (цифры, имена, сроки)
   → Списки и таблицы (разбивают монотонность)

4. КУЛЬМИНАЦИЯ
   → Главный инсайт / вывод
   → «Бонус» — дополнительная ценность

5. ФИНАЛ
   → Краткие выводы
   → Вопрос для комментариев
   → Призыв к подписке
```

### 5.2. Приёмы удержания

| Приём | Пример | Эффект |
|-------|--------|--------|
| **Открытая петля** | «Но главная ошибка — не в этом. О ней расскажу ниже» | +15-20% дочитываний |
| **Буллиты и списки** | Вместо сплошного текста | +10% дочитываний |
| **Подзаголовки-вопросы** | «А что, если бюджет ограничен?» | +5-10% дочитываний |
| **Микроистории** | «Один из наших клиентов потратил 3 месяца на...» | +10-15% дочитываний |
| **Визуал** | Скриншоты, схемы, инфографика | +10% дочитываний |
| **Прямая речь** | «Как сказал CTO компании X: ...» | +5% дочитываний |

### 5.3. Убийцы дочитываний

- Длинные абзацы (> 4-5 предложений)
- Академический стиль
- Отсутствие визуала
- Вводные слова и «вода»
- Монотонная структура
- Раздутые статьи, выходящие за 3500 символов без пробелов
- Несоответствие заголовка содержанию

## 6. Типы статей для Дзен

### 6.1. Образовательная статья

**Формат:** «Что такое X и зачем это бизнесу»
**Объём:** 2500-3500 символов без пробелов
**Структура:**
1. Крючок (проблема / вопрос)
2. Определение простым языком
3. Зачем нужно (с цифрами)
4. Как работает (пошагово)
5. Примеры из практики
6. Инструменты (где VK WorkSpace вписывается органично)
7. Выводы + вопрос

### 6.2. Практический гайд

**Формат:** «Как сделать X: пошаговая инструкция»
**Объём:** 2500-3500 символов без пробелов
**Структура:**
1. Проблема + кому подходит инструкция
2. Что понадобится (чек-лист)
3. Шаг 1-N с деталями
4. Частые ошибки
5. Результат (что получится)
6. Выводы + вопрос

### 6.3. Сравнение / обзор

**Формат:** «X vs Y: что выбрать в 2026 году»
**Объём:** 2500-3500 символов без пробелов
**Структура:**
1. Зачем сравниваем (контекст)
2. Критерии сравнения
3. Обзор решения 1
4. Обзор решения 2 (и далее)
5. Сравнительная таблица
6. Для кого что подходит
7. Выводы + вопрос

### 6.4. Кейс / история

**Формат:** «Как компания X решила проблему Y»
**Объём:** 2500-3500 символов без пробелов
**Структура:**
1. Ситуация: кто, что, зачем
2. Проблема: что пошло не так
3. Решение: что сделали
4. Результат: цифры, метрики
5. Уроки: что извлекли
6. Выводы + вопрос

### 6.5. Подборка / рейтинг

**Формат:** «N лучших инструментов для...»
**Объём:** 2500-3500 символов без пробелов
**Структура:**
1. Критерии отбора
2. Обзор каждого инструмента (3-5 предложений + плюсы/минусы)
3. Сравнительная таблица
4. Рекомендации по выбору
5. Выводы + вопрос

### 6.6. Тренд-статья

**Формат:** «N трендов в ... на 2026 год»
**Объём:** 2500-3500 символов без пробелов
**Структура:**
1. Почему тренды важны
2. Тренд 1: описание + данные + пример
3. Тренд 2-N: аналогично
4. Как подготовиться
5. Выводы + вопрос

### 6.7. Мнение / колонка

**Формат:** Экспертный взгляд на проблему
**Объём:** 2500-3500 символов без пробелов
**Структура:**
1. Тезис (провокационный)
2. Аргумент 1 + пример
3. Аргумент 2 + пример
4. Контраргумент (честность)
5. Вывод
6. Вопрос для дискуссии

## 7. Правила интеграции VK WorkSpace в статьи Дзен

### 7.1. Принципы

- **Органичность:** VK WorkSpace упоминается как один из инструментов, не как единственное решение
- **Через сценарий:** не «VK WorkSpace — лучшая платформа», а «для решения этой задачи подойдёт корпоративная платформа, например VK WorkSpace»
- **Количество:** 2-4 упоминания на статью (мягче, чем в SEO-статьях)
- **Без прямой рекламы:** нет «купите», «попробуйте бесплатно» (Дзен наказывает за промо)
- **Конкуренты:** можно упоминать честно, если тема подразумевает сравнение

### 7.2. Допустимые CTA для Дзен

| CTA | Когда использовать |
|-----|-------------------|
| «Подписывайтесь на канал — будет ещё много полезного» | Всегда в конце |
| «Расскажите в комментариях, какие инструменты используете» | Для вовлечения |
| «Если тема интересна — ставьте лайк, разберём подробнее» | Для вовлечения |
| «Подробнее о платформе — на workspace.vk.ru» | Ненавязчивая ссылка |
| «Пишите в комментариях — ответим на вопросы» | Для дискуссии |

### 7.3. Недопустимые CTA для Дзен

- «Попробуйте бесплатно 30 дней» (промо — риск бана)
- «Запросите демо» (прямая продажа)
- «Переходите по ссылке и оформите подписку» (агрессивный CTA)
- «Скидка / акция / промокод» (реклама)

## 8. Требования к визуальному контенту

### 8.1. Обложка статьи

- Размер: 1200x628 px (рекомендуемый)
- Яркая, контрастная, читаемая на мобильных
- Текст на обложке: крупный, до 5-7 слов
- Не используй стоковые фото — они снижают CTR
- Схемы, инфографика, скриншоты > стоковые фото
- Лица людей повышают CTR на 15-20%

### 8.2. Изображения в статье

- 3-5 изображений на статью
- Скриншоты интерфейсов (с пояснениями)
- Схемы и диаграммы (понятные без текста)
- Инфографика (факты в визуальном формате)
- Подписи к изображениям обязательны
- Alt-текст для доступности

## 9. Модерация и ограничения Дзен

### 9.1. Запрещённый контент

- Прямая реклама без маркировки
- Кликбейт (несоответствие заголовка)
- Копипаст из других источников
- Шоковый контент
- Введение в заблуждение
- Политическая агитация

### 9.2. Ограничения для коммерческого контента

- Рекламные материалы требуют маркировки (Реклама, ООО «...», ИНН, erid)
- Нативная реклама допускается, но с маркировкой
- Экспертный контент с мягким упоминанием продукта — допускается БЕЗ маркировки, если:
  - Основная ценность статьи — в экспертизе
  - Продукт упоминается как один из примеров
  - Нет прямых призывов к покупке
  - Статья полезна без знания продукта

### 9.3. Карма и репутация канала

- Карма канала зависит от качества контента
- Нарушения снижают карму → снижаются охваты
- Регулярность публикаций повышает карму
- Высокие дочитывания повышают карму

## 10. Тон и стиль для Дзен

### 10.1. Характеристики

- **Разговорный** — как будто объясняешь коллеге за обедом
- **Конкретный** — цифры, примеры, имена
- **Эмоциональный** — допускается лёгкий юмор, личные наблюдения
- **Экспертный** — но без академизма
- **Честный** — признаём сложности, не идеализируем

### 10.2. Сравнение стилей

| Параметр | SEO-статья (блог) | Дзен |
|----------|-------------------|------|
| Тон | Профессиональный, консультативный | Разговорный, живой |
| Абзацы | 3-4 предложения | 1-3 предложения |
| Подзаголовки | Содержат ключевые слова | Содержат интригу/вопрос |
| Ключевые слова | Приоритет (SEO) | Вторичны (вовлечение) |
| CTA | «Попробуйте бесплатно» | «Подпишитесь / комментируйте» |
| Визуал | Опционален | Обязателен (3-5 изображений) |
| Длина | 1500-3000 слов | 1000-2500 слов |
| Метрика успеха | Позиция в поиске | Дочитывания, лайки, подписки |

### 10.3. Запрещённые приёмы

- «В современном мире...»
- «Как известно...»
- «Следует отметить...»
- Канцеляризмы: «является», «осуществляется», «в рамках»
- Академический стиль
- Пресс-релизный тон
- «Уникальный», «инновационный», «передовой»

## 11. SEO-элементы в статьях Дзен

Хотя Дзен — рекомендательная платформа, SEO-элементы помогают попасть в поиск Яндекса:

### 11.1. Базовая SEO-оптимизация

- Заголовок содержит ключевую фразу (естественно, без спама)
- Описание статьи содержит ключевую фразу
- H2 содержат вариации ключевых фраз
- Первый абзац содержит основную тему
- Нет переспама ключами (алгоритм Дзен наказывает)

### 11.2. Работа с поисковыми запросами из ТЗ

Если в ТЗ указаны поисковые запросы, соблюдай правила:

```yaml
search_queries:
  minimum_usage: 1  # каждый запрос минимум 1 раз в тексте
  distribution: equal  # равномерно по тексту, не скапливать
  in_headings: true  # по возможности использовать в H2
  natural: true  # вхождения естественные, не в ущерб читаемости
  priority: secondary  # дочитывания важнее ключей
```

**Пример распределения для 4 запросов в статье с 4 H2:**
- Запрос 1 → H2-1 или лид
- Запрос 2 → H2-2 или тело секции 2
- Запрос 3 → H2-3 или тело секции 3
- Запрос 4 → H2-4 или заключение

### 11.3. Приоритет метрик

```
Дочитывания > CTR > Вовлечение > SEO-ключи
```

SEO в Дзен — это бонус, а не основа стратегии.

## 12. Метрики успешности статьи

### 12.1. Пороговые значения

| Метрика | Плохо | Средне | Хорошо | Отлично |
|---------|-------|--------|--------|---------|
| CTR | < 1% | 1-3% | 3-7% | > 7% |

### 12.2. Чеклист качества статьи для Дзен

**Заголовок и описание:**
- [ ] Заголовок: если из ТЗ — использован дословно; если свободный — до 56 символов
- [ ] Описание: дополняет заголовок, до 200 символов

**Объём и структура:**
- [ ] Объём: 2500-3500 символов без пробелов (допуск ±10-20%)
- [ ] Заголовков H2: не более 5
- [ ] Лид: цепляет с первого предложения (боль/вопрос/факт)
- [ ] Абзацы: не более 3 предложений
- [ ] Подзаголовки: интригующие (вопросы, утверждения)

**Поисковые запросы (если указаны в ТЗ):**
- [ ] Каждый запрос использован в тексте минимум 1 раз
- [ ] Запросы распределены равномерно по тексту
- [ ] Запросы использованы в H2 (где возможно)

**Контент:**
- [ ] Визуал: 3-5 изображений (описаны в alt-текстах)
- [ ] Примеры: конкретные, с цифрами
- [ ] VK WorkSpace: 2-4 органичных упоминания (без промо)
- [ ] Нет кликбейта: заголовок соответствует содержанию
- [ ] Нет рекламы: отсутствуют призывы к покупке
- [ ] CTA: вопрос для комментариев + призыв к подписке
- [ ] Нет «воды»: каждое предложение несёт смысл
- [ ] Нет штампов: «в современном мире», «следует отметить»
- [ ] Читаемость: простой язык, короткие предложения

## 14. Режимы генерации для Дзен

### Engagement (по умолчанию)
Фокус на максимальные дочитывания и вовлечение. Разговорный стиль, короткие абзацы, частые подзаголовки, открытые петли.

### Expert
Экспертная статья с глубоким погружением. Больше данных и аналитики, но сохраняется живой стиль Дзен.

### Light
Лёгкая и доступная статья. Минимум терминологии, много примеров, акцент на эмоции и личный опыт.

### Story
Нарративный формат — через историю и сторителлинг. Минимум списков, максимум повествования.
