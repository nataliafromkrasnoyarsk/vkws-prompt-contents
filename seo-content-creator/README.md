# SEO Content Creator Plugin v6.0

Интерактивный агент для создания SEO-статей с **кнопками выбора**, пошаговым заполнением параметров, выбором роли эксперта, доработкой результата и гибким сохранением.

## Что делает плагин

Проводит пользователя через **11-шаговый wizard** с интерактивными кнопками для выбора опций. Вместо ввода номеров — нажимайте кнопки для выбора эксперта, режима, тона, аудитории и других параметров. Поддерживает выбор экспертной роли автора, ввод собственной информации, доработку результата и сохранение в различных форматах.

## Ключевые особенности v6.0

- **Интерактивные кнопки** — выбор опций нажатием кнопок вместо ввода текста
- **Формат кнопок**: `[Текст](claude:команда)` — стандартный синтаксис Claude Code
- **4 эксперта** — ИБ, VMware, Kubernetes, универсальный
- **11 шагов** — от темы до сохранения
- **Доработка** — изменение тона/аудитории после генерации

## Быстрый старт

### Автоматический запуск

Просто напишите одну из фраз:
- "Создай статью"
- "Напиши статью"
- "Новая статья"
- "SEO статья"

### Через команду

```bash
/seo-content-creator:create-article
```

или глобально:

```bash
/seo-create-article
```

## Эксперты

Доступные роли экспертов:

| Эксперт | Описание |
|---------|----------|
| `vk-cloud-security` | Эксперт по ИБ VK Cloud (сертификации, compliance, защита инфраструктуры) |
| `virtualization-vmware` | Эксперт по виртуализации и VMware (vSphere/vSAN/NSX, миграция, российские платформы, VK Cloud) |
| `kubernetes-vk-cloud` | Эксперт по Kubernetes VK Cloud (Cloud Containers, контейнеризация, DevOps, CI/CD, Service Mesh, GitOps) |
| `general` | Универсальный автор (по умолчанию) |

Эксперты определяют терминологию, глубину и стиль статьи.

## Доработка результата

После генерации можно доработать статью (Step 10):

| Опция | Описание |
|-------|----------|
| `всё отлично` | Перейти к сохранению |
| `изменить тон` | Перегенерировать в другом тоне |
| `изменить аудиторию` | Перегенерировать для другой аудитории |
| `короче` | Сократить статью |
| `подробнее` | Расширить статью |
| `упростить` | Сделать проще для понимания |
| `добавить примеры` | Добавить больше примеров |
| `свои правки` | Указать конкретные изменения |

## Сохранение результата

После доработки статьи доступны варианты сохранения (Step 11):

| Опция | Описание |
|-------|----------|
| `markdown` | Сохранить в .md файл |
| `html` | Сохранить в .html файл |
| `content-hub` | Сохранить в content-hub/drafts/ |
| `clipboard` | Скопировать в буфер обмена |
| `все` | Сохранить во все форматы |
| `не сохранять` | Пропустить (по умолчанию) |

## Навигация

В любой момент можно использовать команды:

| Команда | Действие |
|---------|----------|
| `назад` | Вернуться на шаг назад |
| `пропустить` | Пропустить опциональный шаг |
| `статус` | Показать текущие параметры |
| `начать заново` | Перезапустить wizard |
| `помощь` | Показать справку |
| `отмена` | Отменить и выйти |

## Параметры

### Режимы (Step 5)

| Режим | Описание |
|-------|----------|
| `balanced` | Стандартный (по умолчанию) |
| `deep_expert` | Технический экспертный |
| `light` | Упрощённый для всех |
| `strict_seo` | Максимум SEO |
| `brand_heavy` | Акцент на бренд |

### Тон (Step 6)

| Тон | Описание |
|-----|----------|
| `expert` | Профессиональный |
| `neutral` | Нейтральный |
| `technological` | Технический |

### Аудитория (Step 6)

| Сегмент | Описание |
|---------|----------|
| `SMB` | Малый/средний бизнес |
| `enterprise` | Крупный бизнес |
| `developers` | Разработчики |
| `marketers` | Маркетологи |
| `general` | Широкая аудитория |

### Ограничения (Step 8)

- `no_promo` — без рекламы
- `no_comparison` — без сравнений
- `no_hype` — без преувеличений
- `no_pricing` — без цен
