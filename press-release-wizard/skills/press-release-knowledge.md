## КЛЮЧЕВЫЕ ПРАВИЛА НАПИСАНИЯ

### 1. БЕЗ БУЛЛИТОВ В ТЕКСТЕ РЕЛИЗА

Текст пресс-релиза пишется сплошным текстом. Маркированные списки запрещены.

**НЕПРАВИЛЬНО:**
```
Сервис позволяет:
• проверять код на уязвимости
• анализировать зависимости
• генерировать отчёты
```

**ПРАВИЛЬНО:**
```
Сервис проверяет код на уязвимости, анализирует зависимости
и генерирует отчёты для compliance-аудитов.
```

### 2. РАСШИФРОВКА ТЕРМИНОВ ДЛЯ ЖУРНАЛИСТОВ

Пресс-релиз читают журналисты, а не инженеры. Сложные термины нужно объяснять.

**Словарь расшифровок:**

| Термин | Расшифровка в тексте |
|--------|---------------------|
| CI/CD | процесс автоматической сборки и доставки кода |
| DevSecOps | практика встраивания проверок безопасности в разработку |
| DevOps | методология объединения разработки и эксплуатации |
| Kubernetes | система для управления контейнерами |
| Контейнер | изолированное приложение с собственным окружением |
| API | программный интерфейс для интеграции |
| SDK | набор инструментов для разработчиков |
| Zero Trust | модель безопасности с проверкой каждого запроса |
| SLA | гарантированный уровень доступности |
| On-Premise | установка на серверах клиента |
| Public Cloud | публичное облако с общей инфраструктурой |
| Private Cloud | частное облако для одной организации |
| ML/Machine Learning | машинное обучение |
| ETL | процесс извлечения, преобразования и загрузки данных |
| In-memory | обработка данных в оперативной памяти |

**Формат расшифровки в тексте:**
```
Сервис интегрируется в CI/CD-пайплайн — процесс автоматической
сборки и доставки кода — и проверяет каждый коммит.
```

### 3. ОПИСАНИЕ ПРОДУКТА ПОСЛЕ ЛИДА

После лид-абзаца обязательно идёт описание продукта (1-2 абзаца сплошным текстом):
- Что делает продукт (функционал)
- Как это работает (технология простыми словами)
- Для кого предназначен (аудитория)
- Варианты использования (сценарии)

## ЗАГОЛОВКИ

### Формула

```
[Компания] + [глагол действия] + [продукт/решение] + [ценность/аудитория]
```

### Глаголы для заголовков

| Контекст | Глаголы |
|----------|---------|
| Новый продукт | запустила, представила, выпустила |
| Обновление | обновила, расширила, улучшила |
| Партнёрство | и [партнёр] запускают |
| Линейка | выводит на рынок, представила линейку |
| Достижение | разработала, создала |

### Примеры из VK Cloud

```
✓ VK Cloud представила облачный сервис для создания виртуальных рабочих мест

✓ QazCloud и VK Cloud запускают локализованную облачную платформу в Казахстане

✓ VK выводит на рынок собственные продукты информационной безопасности

✓ VK Tech разработал единую платформу для работы с данными

✓ Платформа VK Cloud упростит разработчикам создание цифровых продуктов
```

### Антипаттерны

```
✗ VK Cloud — инновационное решение для бизнеса (нет действия)
✗ Анонс нового революционного продукта от VK (превосходные степени)
✗ VK Cloud Solutions представляет вашему вниманию... (многословность)
```

## ЛИД-АБЗАЦ

### Структура

```
Предложение 1: [Компания] [действие] [продукт] — [Название].
Предложение 2: [Главная ценность для пользователя].
Предложение 3: [Технологическая база / партнёр / география].
```

### Примеры

**Запуск продукта:**
```
VK Cloud запустила облачный сервис для создания виртуальных 
рабочих мест — Cloud Desktop. Он упростит работу распределенных 
команд и организацию доступа удаленных сотрудников к корпоративным 
системам. Сервис работает на базе решения для удаленного доступа 
Termidesk компании «Увеон — облачные технологии» (входит в «Группу Астра»).
```

**Партнёрство:**
```
Облачная платформа для цифровизации бизнеса VK Cloud развернута 
в Казахстане в партнерстве с QazCloud. Технологии и сервисы VK Cloud 
помогут как крупному, так и малому бизнесу и стартапам развивать 
цифровые проекты, быстрее выводить их на рынок и гибко масштабировать. 
Облачная платформа адаптирована для компаний Казахстана и размещается 
в дата-центре QazCloud на территории республики.
```

**Линейка продуктов:**
```
На конференции VK Cloud Conf 2025 компания VK Tech представила 
линейку новых сервисов информационной безопасности, разработанных 
и протестированных в VK. Сервисы становятся частью продуктовой 
экосистемы VK и интегрируются с облачной платформой VK Cloud 
во всех вариантах поставки.
```

## ОПИСАНИЕ ПРОДУКТА

### Что включать

1. **Функционал** — что делает продукт
2. **Аудитория** — кому предназначен
3. **Сценарии** — конкретные примеры использования
4. **Варианты** — типы или комплектации
5. **Технология** — на чём построен (если важно)

### Пример описания

```markdown
С помощью Cloud Desktop бизнес может организовать виртуальные 
рабочие места под разные подразделения компании. Сессионные 
подходят для больших команд, где у сотрудников похожие задачи 
и потребности в программном обеспечении — например, в колл-центрах. 
Персонализированные учитывают индивидуальные требования к рабочему 
месту и необходимым для сотрудника инструментам. Подключиться 
к виртуальному рабочему месту можно с любого устройства.
```

### Описание нескольких продуктов

```markdown
Всего было представлено четыре решения.

Среди них — платформа для автоматизации проверки безопасности 
кода **Security Gate**. Она обеспечивает безопасность разработки 
для всех приложений и более 40 000 репозиториев VK, проводит 
более 10 000 сканирований в день и охватывает более 1 000 000 000 
строк кода.

Для оперативного реагирования на инциденты безопасности разработана 
система **SIEM**. Она способна обрабатывать до 10 000 000 событий 
в секунду и хранить до 15 ПБ телеметрии.

Обеспечивать безопасность каждого запроса помогает сервис **ZTA** 
(Zero Trust Architecture). Сервис позволяет снижать риски кражи 
паролей и выявлять аномалии в поведении пользователей.

Также компания презентовала систему защиты данных **DSPM**, 
которая показывает, где находятся конфиденциальные данные и кто 
имеет к ним доступ.
```

```yaml
1_market_context: "Тренд или ситуация на рынке"

3_solution: "Как продукт решает проблему"
4_value: "Конкретная выгода с цифрами"
```

```markdown
*«{текст цитаты}»*, — комментирует **{Имя Фамилия}, {должность}**.
```

или


## ЦИФРЫ И МЕТРИКИ

### Как оформлять

```yaml
growth:
  - "выросла на 62% год к году"
  - "увеличилась в 2,2 раза"
  - "рост на 82% по сравнению с 2023 годом"

performance:
  - "до 10 000 000 событий в секунду"
  - "более 1 000 000 000 строк кода"
  - "до 15 ПБ телеметрии"
  - "время отклика до 30 миллисекунд"

savings:
  - "сократить затраты до 60%"
  - "за несколько минут" (вместо дней/недель)
  - "без дополнительных инвестиций"

coverage:
  - "более 20 преднастроенных сервисов"
  - "более 40 000 репозиториев"
  - "более 100 продуктов и сервисов"

market:
  - "больше 95% аудитории рунета"
  - "топ-3 поставщиков ИТ-решений в России"
```

### Правила

```yaml
rules:
  - Минимум 2 цифры на релиз
  - Цитата спикера должна содержать хотя бы 1 цифру
  - Сравнение год к году / квартал к кварталу
  - Указывать источник для рыночных данных
```

```yaml
  - "революционный"
  - "уникальный"
  - "не имеющий аналогов"
  - "лучший на рынке"
```

## ВАЛИДАЦИЯ ЦИФР

### Автоматическая проверка

Перед выдачей релиза проверь наличие цифр:

```yaml
minimum_numbers: 2
quote_numbers: 1  # минимум в цитате

number_patterns:
  - процент: "на X%", "X% год к году"
  - количество: "более X", "до X", "X тысяч/млн/млрд"

  - сравнение: "в X раз", "с X до Y"

```

```
Варианты для усиления:

- Масштаб: "используется в X компаниях"

Добавить цифры или оставить как есть?
```

## SEO-ЭЛЕМЕНТЫ

### Meta-данные для публикации

При запросе "экспорт html" или "seo" генерируй:

```yaml
seo:
  title: "{Заголовок релиза} | {Компания}"
  description: "{Первые 150-160 символов лид-абзаца}"
  keywords:
    - "{продукт}"
    - "{компания}"
    - "{категория: облачные сервисы / ИБ / разработка}"
    - "пресс-релиз"
  og_title: "{Заголовок}"
  og_description: "{Лид-абзац сокращённый до 200 символов}"
  og_type: "article"
```

### Примеры SEO-данных

```yaml

seo:
  title: "VK Cloud запустила сервис для повышения безопасности приложений | VK Cloud"

  keywords:
    - "Cloud Audit"
    - "VK Cloud"
    - "DevSecOps"
    - "безопасность кода"
    - "облачные сервисы"
```

### Ключевые слова для заголовков

```yaml
high_priority_keywords:
  - название продукта
  - название компании
  - тип решения (платформа, сервис, система)

medium_priority_keywords:
  - целевая аудитория
  - ключевая технология
  - география (если релевантно)
```
