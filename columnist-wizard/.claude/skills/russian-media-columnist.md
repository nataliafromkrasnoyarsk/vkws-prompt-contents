## PART 1: BUSINESS PUBLICATIONS (ДЕЛОВЫЕ СМИ)

### Forbes Russia (forbes.ru)

**Publication Profile**
- Status: Active (owned by independent Russian investor since 2019)
- Audience: C-level executives, investors, business leaders
- Founded: 2004 (licensed Axel Springer edition)
- Format: Website + print presence

**Column Requirements**

| Aspect | Requirement |
|--------|-------------|
| Length | 800-1,200 words optimal |
| Tone | Authoritative, data-driven, strategic |
| Technical depth | 20-30% |
| Publishing frequency | 2-5 posts/month for active contributors |
| Self-promotion | STRICTLY FORBIDDEN |

**Structure Requirements**:
- Headline: 8-12 words, compelling and specific
- Opening: 2-3 sentences that hook immediately
- Body: 3-5 sections with H2 subheadings
- Evidence: 2-3 external sources cited
- Closing: Actionable takeaways

**Critical Rules**:
- NO product promotion (primary rejection reason)
- NO "how to buy" guidance
- NO favorable competitor comparisons
- Must disclose affiliations
- Data-backed claims only

**Author Bio Format**:
```
[Имя Фамилия] — [должность] в [компания], которая [краткое описание].
[Компания] конкурирует с [упомянутые компании] в данной статье.
```

**What Works**:
- Contrarian perspectives with evidence
- Trend analysis with Russian market angle
- Strategic frameworks for decision-makers
- Industry predictions backed by data
- Executive experience insights (anonymized)

## PART 2: TECHNOLOGY PUBLICATIONS (ТЕХНОЛОГИЧЕСКИЕ СМИ)

### Habr / Хабр (habr.com)

**Publication Profile**
- Format: Corporate blogs (Корпоративные блоги)
- Audience: Developers, architects, tech professionals
- Style: Technical, authentic, no marketing
- Volume: 9,000+ corporate articles annually

**Column Requirements**

| Aspect | Requirement |
|--------|-------------|
| Length | 1,500-2,500 words (8,000+ chars without spaces) |
| Tone | Technical but authentic, conversational |
| Technical depth | 60-80% |
| Format | Markdown with `<cut />` tag |
| Code examples | Mandatory for tutorials |

**Structure Requirements**:
- Compelling headline
- Preview text: 500-1,500 characters before `<cut />`
- Table of contents for long articles
- Subheadings dividing logical sections
- Code blocks with syntax highlighting
- Visual elements: diagrams, screenshots

**Community Standards**:
- NO press-release style (actively disliked)
- NO marketing rhetoric
- Honest about limitations
- Open to discussion and criticism
- Practical value essential

**Formatting Requirements**:
- Proper typography: em-dashes (—), curly quotes («»), "ё" usage
- Numbered and bulleted lists
- Tables and infographics
- Code syntax highlighting
- Short paragraphs (2-4 sentences)

**What Works**:
- Practical problem-solving
- Real-world technical challenges
- Original methodology or approach
- Honest assessment of trade-offs
- Engaging discussion topics
- Controversial takes (sparks engagement)

**What Fails**:
- Marketing-speak
- Generic content
- Superficial technical depth
- No practical value
- Press-release tone

## PART 4: EDITORIAL RELATIONSHIPS & PITCHING

### Pitch Email Template (Russian)

```
Тема: Pitch: [Конкретный, цепляющий заголовок]

Здравствуйте, [Имя редактора],

[Личная связь, если есть — видел вашу статью о X, встречались на конференции Y]

Предлагаю колонку на тему [ТЕМА]:

**Заголовок**: [Рабочий заголовок — 8-12 слов]
**Угол**: [Что нового/отличного/актуального]
**Почему сейчас**: [Привязка к тренду, событию, данным]
**Почему я**: [Ваши credentials и уникальная перспектива]
**Что получит читатель**: [Ключевые takeaways и ценность]

Формат: [Мнение/анализ/кейс] — [примерный объём]

Ключевые тезисы:
- Тезис 1
- Тезис 2
- Тезис 3

Могу подготовить черновик в течение [срок]. Подойдёт ли это для [Издание]?

С уважением,
[Имя]
[Должность, Компания]
[LinkedIn, предыдущие публикации]
```

### Common Pitch Mistakes

| Mistake | Better Approach |
|---------|-----------------|
| "Хочу написать про облака" | "Анализ: почему 3 из 5 крупнейших сбоев облаков имели общую архитектурную ошибку" |
| "Наш продукт отличный и..." | "CIO, борющиеся с X, могут использовать подход Y, который снизил наши затраты на 60%" |
| "Думаю, это было бы интересно" | "По данным [источник], 67% компаний сталкиваются с [проблема] — вот что показало наше исследование" |
| Без привязки к изданию | "Учитывая фокус [Издание] на [тема], это хорошо вписывается в..." |

## PART 5: RUSSIAN LANGUAGE STANDARDS

### Register (Деловой стиль)

**Business Publications (Forbes, RBC, Vedomosti, Kommersant)**:
- Use polite, formal forms
- Avoid colloquial expressions
- Prefer complete word forms over abbreviations
- Professional vocabulary throughout
- "Вы" form always

**Technical Publications (Habr, VC.ru)**:
- More conversational acceptable
- Technical slang permitted
- "Вы" form standard, occasional "ты" in community context

### Technical Terminology

**Established Russian Terms**:
- контейнер (container)
- кластер (cluster)
- развернуть / деплой (deploy)
- отказоустойчивость (fault tolerance)
- масштабирование (scaling)
- облачные вычисления (cloud computing)
- виртуализация (virtualization)
- инфраструктура как код (IaC)

**Accepted English Borrowings**:
- docker, kubernetes, git, API, REST
- DevOps, DevSecOps, CI/CD
- machine learning, AI
- SaaS, PaaS, IaaS

**Format for Mixed Terms**:
- First use: "контейнеризация (containerization)"
- Subsequent: "контейнеризация"
- Acronyms: Full form first, then abbreviation

### Sentence Structure

**Russian allows longer sentences**:
- 15-35 words acceptable (vs. 15-20 for English)
- Vary length for rhythm
- Known → new information order
- Avoid complex nested constructions

**Typography Requirements**:
- Em-dashes: — (not -)
- Quotes: «кавычки» (not "quotes")
- Letter ё: use consistently
- No double spaces
- Non-breaking spaces before: %, №, units

### Quality Checklist

Before submission to any Russian publication:

- [ ] Technical claims verified
- [ ] Code examples tested (if applicable)
- [ ] Statistics sourced and current
- [ ] Grammar and spelling perfect (spellcheck used)
- [ ] Active voice predominant
- [ ] Jargon defined or avoided
- [ ] Terminology consistent
- [ ] Tone appropriate to publication
- [ ] Affiliations disclosed
- [ ] Conflicts declared
- [ ] Meta description written (150-160 chars)
- [ ] Author bio included
